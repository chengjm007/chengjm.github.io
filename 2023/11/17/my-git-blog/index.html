对于卡片盒笔记写作法的三点认识

发表于 2023-11-17 更新于 2023-11-25 分类于 兴趣领域 ， Obsidian ， 笔记方法 阅读次数： 本文字数： 2.5k 阅读时长 ≈ 2 分钟
我的第一篇 Git Blog。一篇介绍读书笔记的笔记，让你重新认识写作的流程。

《卡片笔记写作法》

介绍了德国彼得菲尔德大学社会学教授尼克拉斯.卢曼高效产出的写作方法，这个方法的核心是卡片盒，开始以为就是很简单的一种方法，所以第一次读完这本书并没有给我留下很特别的印象，以为就是写个笔记卡片，然后以一定的规则编好号，在写作成稿时再按编号顺序串起来就行了。

书中介绍了根据编号产生的笔记卡片之间的互相链接，也就是“双链”。现在的电子笔记很多都有这个功能，比如 Notion，确实能实现类似“套娃”的无限链接功能，很方便内容之间的即时跳转，跟 Wiki 一样的。但 Notion 网络响应过慢，而且还要绑定 Apple ID 登录，整体感觉“太重”，不怎么喜欢用，但作为知识管理特别是项目管理绝对是首选。

书中特别提到了能很好实现“双链”功能的电子笔记工具 Obsidian，所以我专门下载试用。折腾了好一阵子，总算摸到了一点应用的皮毛，现在能够使用基本的 Markdown 语法写笔记，会套用 Template 模版，反复试用了近 40个插件，会一点简单的 Dateview 和 YAML 语法。笔记记了 166 篇，关系图谱也很直观，能看到笔记之间的链接，但不知道怎么把这个关系图谱用起来，“双链”的价值仅限于展示。

说实话，Obsidian 码字确实非常丝滑，而且永久免费、数据本地存储、适用办公内网离线部署，完美解决我的个性化需求。相比 WPS 更“轻”，启动更快，简直让我爱上了码字，几乎重新燃起了我尘封多年的记忆，总想写点什么。一口气复制粘贴了几十篇内网网页工作信息当作文献笔记，也加进了标签和笔记链接。但是写着写着，总觉得哪里不对劲。书中非常推崇笔记之间的“双链”，但真正实践起来却并知道怎么去发挥“双链”的作用，仅仅只能从关系图谱上可视化关系图谱11:10，但并不能给自己带来生产力的提升，现有的笔记只是一种存储而已。

于是，我又重新去读这本书，希望能再厘清这个笔记方法的真谛，但是书只重读了一半，收获甚微，理解能力仍然有限。只能上 B 站找相关的视频反复学习，学终有获，总算弄清一点眉目。

首先，我们要搞清做笔记的目的到底是什么？

之前我想当然地以为做笔记就是“好记性抵不住一个烂笔头”，笔记不就是用来储存以备将来不时之需嘛。此想法根深蒂固，但大错特错，做笔记是为了输出的，是要将自己积累的知识传播出去的。这个观点，得益于付费参与笔记实训营清单控沙牛的指导，后来看到的很多 U 主都强调了这一观点，虽然表述不一，但是意思相同。自己以前使用的印象笔记、有道笔记，其实早已成了笔记的坟场，所有收藏的文章和记录的感想，都生下就“吃灰”了，即使想用起来但只能在笔记里重新再翻一次，基本都记不起来了。

其次，卡片盒笔记法不是一个方法，而是一个系统，一个写作系统（流程）。

一般我们写作的方法，先搭好提纲，然后一二三四五谋篇布局，需要预先设计好一个整体框架思路，再绞计脑汁沉思闷想，逐段推进，基本是一种画组织机构图或思维导图的模式展开，属于自上而下的套路。但是，卡片盒笔记法不是这样，而是恰恰相反，是自下而上6:50的写作顺序。这个“自下”，就是自每一张张的小卡片，单个一张的小卡片是发挥不了很大作用的，但多个卡片组合在一起的卡片组就能显现出威力了，它能通过互相链接的对接, 自然而然碰撞产生一种新的关系，有时候还会让你收获意想不到的灵感和思维的火花。随着笔记数量和链接的增多，关系图谱会像大树根一样到处生根发芽，就会很自然地从单个的笔记种子逐渐长成为有树干和分支的大树。

第三，卡片盒笔记法不等于卡片笔记法，一字之差，内涵迥异。

卡片盒笔记法的自下而上写作流程，也可以理解为“自小而大”，“自小”，指写作是从一张一张的小卡片做起的，并不需要一开始就布局宏篇巨著，这样就避免了传统写作过程中遇到的思路卡壳受阻障碍。因为一张卡片只记录一个知识点，每个知识点基本不超过 5 条，可以做到信手拈来毫不费力。当成千上万张卡片通过不同逻辑的组合链接在一起时，会自然产生“文思泉涌 ”的效果，根本不会出现“黔驴技穷”的苦闷局面。正因为只有卡片组合才能发挥威力，而不是单张卡片的简单接龙，才需要将卡片严格按排列组合规则放入编好号的卡片盒，否则，按原始视频中卢曼的说法，“要把它们摆好，不然问题就大了”。卢曼视频 3:20 所以千万不能忽视了这个装卡片的“盒”，它就是卡片组合的集合。B 站 U 主闻韶同学把卡片组合比喻为打扑克牌，不是取决于你手里有多少张牌，而是看你把这些牌怎么合理组合才能打出“王炸”的效果。扑克牌7:40

第四，关于卡片的分类，需要正本清源。

我认为诠释卡片分类的，还是微信公众号葛仲然讲得更透彻，也是我能记住和理解的。卢曼卡片分为三类：主卡、文献卡、关键词表卡。这里并没有提到闪念笔记卡片。

==主卡==，是内容型卡片，是知识信息的主要载体，主要是卢曼的摘录、评论和想法，主卡比较简单。文献卡和关键词卡是功能型卡片，主要起链接和索引作用。

文献卡相对要复杂一些，大致分为三类：单一文献卡、集合文献卡和大纲文献卡。

单一文献卡，就是只有一条参考文献，但是仍然可以与其它卡片发生链接，也可以看作一张主卡，编号逻辑与主卡相同。

集合文献卡，就是把单一文献卡集合到一起。集合文献卡的编号规则与单一文献卡是不同的，通常是围绕每个关键词或研究主题展开的。为了便于对集合文献卡索引，卢曼专门设计了文献关键词表卡。这个关键词表卡并不是按照 A~Z 顺序排列，而是按照先来后到的顺序自然编排。将参考文献单独成卡，主要还是为了提升引用效率，不过现在有文献管理软件 Zotero 可以方便应用。

大纲文献卡，是针对某个研究主题形成的初始大纲，相当于幕布的作用。这既是列计划，也是思维导图式的头脑风暴。

==关键词表卡==，是卢曼将所有关键词（主题词）归集的地方，这是卡片数据库的总目录，也是唯一的“固定”总入口，是卡片盒笔记法运转的“核心”。找到了关键词表卡，就好比盗墓人找到了皇陵地宫的入口。需要注意的是，三类卡片内容信息都不是一成不变的，而是随研究的深入在不断更新完善补充，终生维护。

总结，卡片盒笔记法分为两个系统：内容系统、索引系统。

内容系统，由主卡组成。索引系统，由卡片编码、文献卡和关键词表卡三部分组成。

所有知识管理的难点不在于如何存储，而在于如何提取。
